# Component Instructions

## General
- Default to Server Components — only add `"use client"` when needed (events, hooks, state)
- Use shadcn/ui primitives from `@/components/ui/` — don't rebuild what exists
- Use `cn()` from `@/lib/utils` for conditional Tailwind classes
- All styling via Tailwind utilities — no inline styles

## Component Locations
- `ui/` — shadcn/ui auto-generated (don't edit manually)
- `kanban/` — Pipeline board: `KanbanBoard`, `KanbanColumn`, `DealCard`
- `ai-chat/` — AI assistant: `ChatSidebar`, `ChatMessage`, `ChatInput`
- `shared/` — App-wide: `AppSidebar`, `PageHeader`, `MetricCard`, `LoadingSkeleton`

## Data Flow
- Server Components fetch data directly via MCP client or async functions
- Client Components receive data as props from Server Component parents
- Mutations go through API routes or Server Actions — never call MCP from client code
