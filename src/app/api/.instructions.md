# API Route Instructions

## Required in Every Route
```typescript
export const runtime = 'nodejs';
```

## Patterns
- Use `NextRequest` and `NextResponse` from `next/server`
- Return JSON responses with `NextResponse.json()`
- Use proper HTTP status codes: 200, 201, 400, 404, 500
- Error shape: `{ error: string, details?: string }`
- Get MCP client via `getMcpClient()` from `@/lib/mcp-client`
- Parse request bodies with `request.json()` and validate with Zod schemas
- MCP tool results come back as `{ content: [{ type: 'text', text: string }] }` — parse the text as JSON

## Important: database_id vs data_source_id
- `NOTION_DB.*` — database IDs, used for `API-post-page` (`parent: { database_id }`)
- `NOTION_DS.*` — data source IDs, used for `API-query-data-source` (`data_source_id`)
- These are DIFFERENT IDs. Import both from `@/lib/notion-schema`

## MCP Tool Names (v2 — API-* naming)
- `API-post-search` — search pages/databases (params: query, filter, sort, start_cursor, page_size)
- `API-query-data-source` — query a database with filters (params: data_source_id, filter, sorts, start_cursor, page_size)
- `API-post-page` — create a new page (params: parent: { database_id }, properties, children)
- `API-patch-page` — update page properties (params: page_id, properties)
- `API-retrieve-a-page` — get a page by ID (params: page_id)
- `API-get-block-children` — get page content blocks (params: block_id)
- `API-retrieve-a-data-source` — get database schema (params: data_source_id)
- `API-create-a-comment` — add comment to a page (params: parent, rich_text)
